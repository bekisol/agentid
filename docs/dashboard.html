Sandbox mode gives you an isolated environment to test agent registration, verification, and signing — with no rate limits, no webhooks fired, and a separate agent table. Perfect for CI/CD pipelines and development.

Checking…
# Sandbox quick-start (Python SDK)
client = AgentIDClient.sandbox(api_key="ak_...")
result = client.agents.register(
  name="Test Agent",
  capabilities=["sign", "verify"],
)

Sandbox agents use the did:sandbox: prefix and are completely isolated from production. You can reset the sandbox at any time without affecting real agents.
